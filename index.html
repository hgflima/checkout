    Introdução

    A API Mundi Checkout é a forma mais simples e rápida para realizar a integração com os meios de pagamento oferecidos pela MundiPagg. O produto é um solução completa que inclui um formulário onde é possível inserir dados de cartão de crédito e realizar o pagamento sem que a loja tenha acesso aos dados do cartão do portador.

    O processo de integração com o Checkout consiste em 3 passos:

    1. Criar um token único para realizar a transação;
    2. Adicionar o script na página de checkout para exibir um formulário; onde o portador irá inserir os dados do meio de pagamento desejado;
    3. Implementar o recebimento da confirmação do pagamento;

    Assim que o script de integração for adicionado à página da loja, o formulário será criado automaticamente, simplificando o processo de integração. As imagens abaixo fazem um preview de como ficará o visual do processo de checkout:

    Botão de pagamento

    O botão de pagamento é o primeiro contato do cliente com o fluxo de checkout. É através da ação de clique do cliente que o restante do fluxo acontecerá.

    Seleção da forma de pagamento

    Assim que o cliente clicar no botão de pagamento, uma tela se abrirá solicitando a escolha da forma de pagamento - cartão, boleto, transferência online.

    Dados do pagamento

    Uma vez selecionada a forma de pagamento, uma nova tela se abrirá para que o cliente informe os dados referentes à forma de pagamento escolhida.

    Integração

    O primeiro passo para se integrar com a API Mundi Checkout é a geração de um token único para a realização da transação. Esse passo é feito no backend da loja, utilizando um script que fará uma requisição ao serviço Mundipagg solicitando o token.

    Como o token tem validade de 30 minutos, é importante que ele seja gerado na página de checkout da loja, quando o cliente já decidiu efetuar a compra. Caso tenha se passado mais do que 30 minutos desde a geração do token, ele expirará e a loja precisará gerar um novo token. Outros pontos também devem ser observados ao gerar o token:

    1. A solicitação do token deve ser realizada pela página de checkout da loja, página onde o cliente concluir todo o processo de carrinho de compra;
    2. Um novo token deve ser gerado a cada visualização desta página pelo cliente;
    3. A identificação do pedido na loja deve ser único e enviado na solicitação de criação do token, campo OrderReference;
    4. Caso a loja efetue a requisição de criação de um token e informe um número de pedido já existente retornaremos o mesmo token criado na primeira solicitação e o valor será atualizado para o novo valor enviado no campo AmountInCents;
    5. O token retonado pela API deverá ser configurado na tag script descrita na próxima seção;

    A API utiliza uma arquitetura REST e a troca de mensagens é feita através do protocolo HTTP. As mensagens deverão ser enviadas para o endpoint http://checkout.mundipagg.com/{recurso} utilizando o recurso Checkout para geração do token ou o recurso Query para efetuar consultas aos dados do pedido e de suas respectivas transações

    Gerando o token

    A criação da requisição REST deve seguir as seguintes regras:

    1. A requisição a ser enviada deve possuir o campo MerchantKey (chave da loja) no header da requisição.
    2. O objeto TokenRequest deve ser enviado no corpo da requisição.

    Resposta

    Em caso de sucesso, HasError será false e TokenKey conterá o token da sessão

    201 (Created)
Content-Type: application/json
{
    "ErrorCollection": [],
    "HasError": false,
    "RequestKey": "75323255-6ba1-4e92-a19c-fb1e89c9cf2a",
    "Success": true,
    "TokenKey": "ad008abf-7877-4c80-bd0a-21e2cbade216"
}

    Em caso de erro, HasError será true e ErrorCollection conterá a descrição do erro

    400 (Bad Request)
Content-Type: application/json
{
    "ErrorCollection": [
        {
            "ErrorCode": 400,
            "ErrorDescription": "The OrderReference field is required.",
            "ParameterName": "Checkout.DataContract.Request.TokenRequestData.OrderRequest.OrderReference"
        }
    ],
    "HasError": true,
    "RequestKey": null,
    "Success": false,
    "TokenKey": null
}

    ErrorCollection Coleção Coleção com os possíveis erros ocorridos com a requisição
    HasError Boolean Indica se houve algum erro com a requisição

    Adicionando script na página de checkout

    O Mundi Checkout disponibiliza, através de um Java Script, um formulário para que o usuário final insira os dados do cartão de crédito. Para que o formulário seja exibido a loja deve adicionar o Script abaixo em sua página de checkout.

    É importante observar que o Token retornado pela API do Mundi Checkout deve ser utilizado para preencher a propriedade data-token dinamicamente e, após a criação do Token e o processamento do script, será apresentado ao cliente um botão de pagamento com o texto informado na tag data-payment-button-text. A função deste botão de pagamento é carregar o Mundi Checkout sobre a página da loja, onde o cliente deverá seguir o fluxo de pagamento até o frame final de sucesso no processamento da transação de pagamento.

    <script id="boxCheckout"
        type="text/javascript"
        src="http://checkout.mundipagg.com/Scripts/Checkout/checkoutFactory.js"

        data-paymentmethod-creditcard="true"
        data-paymentmethod-boleto="true"
        data-with-user-data="true"
        data-color-panel="#3276C5"
        data-url-logo=null
        data-payment-button-text="Efetuar Pagamento"
        data-merchant-name="Posto na Web"
        data-token="TOKEN"
        data-api-url="http://checkout.mundipagg.com"
        data-opacity="70">
</script>
    Campo Tipo Descrição Obrigatório
    data-paymentmethod-creditcard Boolean Habilita o pagamento por cartão de crédito Sim
    data-paymentmethod-boleto Boolean Habilita o pagamento por boleto Sim
    data-with-user-data Boolean Habilita o formulário completo com dados de usuário para validação dos dados de antifraude Sim
    data-color-panel String Define a cor, em hexadecimal #RRGGBB, para renderização dos painéis e formulários do checkout Sim
    data-url-logo String Url do logo tipo da loja Não
    data-payment-button-text String Texto do botão de pagamento Sim
    data-merchant-name String Nome da loja a ser apresentado no painel do checkout Sim
    data-token String Token gerado na resposta da requisição de criação do Token Sim
    data-api-url String Url da API do MundiCheckout Não
    data-opacity Int Oercentual da opacidade da sombra de fundo do checkout Não
